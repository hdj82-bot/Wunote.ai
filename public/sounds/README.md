# 게이미피케이션 효과음 자산

Wunote.md 558~564행 사양에 정의된 학습자 효과음 5종.

## 현재 자산 (in-repo)

5개 사운드는 **자체 합성된(procedurally synthesized) 단음 / 코드 오디오** 입니다.
원본은 WAV (44.1 kHz / 16-bit / mono) 이며, 동일 파일명의 MP3 는 ffmpeg LAME 인코딩 산출물입니다.

| 파일                                       | 트리거                       | 분위기            | 길이    |
| ------------------------------------------ | ---------------------------- | ----------------- | ------- |
| `correct.wav` / `correct.mp3`              | 오류 0개 수정고 제출         | C5–E5–G5 상승 띵동 | ~0.58 s |
| `badge.wav` / `badge.mp3`                  | 새 배지 획득                 | C-major 코드 + 스파클 | ~0.95 s |
| `error-found.wav` / `error-found.mp3`      | 오류 카드 생성 (분석 결과)   | E5→C5 부드러운 알림 | ~0.56 s |
| `streak.wav` / `streak.mp3`                | 스트릭(연속 학습) 갱신       | 노이즈 크래클 + 처프 | ~0.65 s |
| `level-up.wav` / `level-up.mp3`            | 레벨업                       | C5-E5-G5-C6 + 벨 | ~1.07 s |

## 파일 요건

- 포맷: WAV (44.1 kHz / 16-bit / mono) primary, MP3 (VBR ~165 kbps) optional fallback
- 길이: 0.5 ~ 2 초 (UX — 짧을수록 좋음)
- 라우드니스: 정규화 후 `-14 LUFS` 내외, 최대 피크 `-1 dBTP`
  - `scripts/build-sounds.mjs` 가 ffmpeg `loudnorm=I=-14:LRA=7:TP=-1` 로 자동 정규화

## 자산 재생성 / 재인코딩

```bash
# 1) WAV 합성 (Windows / .NET 만 필요)
npm run sounds:gen

# 2) MP3 인코딩 (ffmpeg 필요 — Mac: brew install ffmpeg, Win: winget install ffmpeg)
npm run sounds:build
```

`sounds:build` 는 동일 디렉토리의 모든 `.wav` 를 동명 `.mp3` 로 인코딩합니다.

## 추가 후 확인

1. 학습자 화면에서 임의의 버튼을 클릭(브라우저 자동재생 정책 대응) 후 다음 시나리오를 점검합니다.
   - 학습 페이지에서 텍스트 분석 → `error-found`
   - 0 오류 수정고 제출 → `correct`
   - 환경설정에서 사운드 OFF 후 어떤 트리거에도 무음이어야 함

## 자동재생 정책

브라우저(Chrome/Safari/Edge)는 사용자 인터랙션이 없으면 오디오 재생을 차단합니다.
`SoundProvider` 는 첫 `click`/`keydown`/`touchstart` 이벤트 후에만 엔진을 초기화하며,
그 이전의 재생 요청은 조용히 무시됩니다.
