# 게이미피케이션 효과음 자산

Wunote.md 558~564행 사양에 정의된 학습자 효과음 5종을 이 디렉토리에 배치합니다.

> **현재 상태:** 5개 mp3 파일은 모두 **무음 placeholder** (208 byte, 32 kbps / 44.1 kHz mono, 약 50 ms 무음 2 프레임)
> 입니다. 실제 사운드 파일은 별도 자산 디렉토리(라이선스 확인 완료된 외부 자산)에서 받아
> 동일한 파일명으로 덮어쓰기만 하면 됩니다. 코드(`lib/sounds.ts`)는 변경할 필요 없습니다.
>
> placeholder 가 들어 있는 동안에도 트리거 → 재생 흐름은 정상 동작하며, 사용자에게는 무음으로
> 들립니다. 운영 배포 전에 실제 사운드로 교체하세요.

## 필요한 파일

아래 파일명을 그대로 유지해야 `lib/sounds.ts` 의 `SOUND_FILES` 매핑이 동작합니다.

| 파일명             | 트리거 시점                | 분위기              |
| ------------------ | -------------------------- | ------------------- |
| `correct.mp3`      | 오류 0개 수정고 제출       | 경쾌한 띵동         |
| `badge.mp3`        | 새 배지 획득               | 짧은 팡파레         |
| `error-found.mp3`  | 오류 카드 생성 (분석 결과) | 부드러운 알림음     |
| `streak.mp3`       | 스트릭(연속 학습) 갱신     | 불꽃 효과음         |
| `level-up.mp3`     | 레벨업                     | 축하 사운드         |

## 파일 요건

- 포맷: MP3 (44.1 kHz / 128 kbps 권장)
- 길이: 0.5 ~ 2 초 (UX 가이드 — 짧을수록 좋음)
- 라우드니스: 정규화 후 `-14 LUFS` 내외, 최대 피크 `-1 dBTP`
- 사운드 라이선스는 상업적 사용 가능 여부를 반드시 확인

## 추가 후 확인

1. 파일을 위 표와 동일한 이름으로 `public/sounds/` 에 둡니다.
2. (선택) `npm install howler @types/howler` — 미설치 시 `SoundManager` 가 native `HTMLAudioElement` 로 폴백합니다.
3. 학습자 화면에서 임의의 버튼을 클릭(브라우저 자동재생 정책 대응) 후 다음 시나리오를 점검합니다.
   - 학습 페이지에서 텍스트 분석 → `error-found.mp3`
   - 0 오류 수정고 제출 → `correct.mp3`
   - 환경설정에서 사운드 OFF 후 어떤 트리거에도 무음이어야 함

## 자동재생 정책

브라우저(Chrome/Safari/Edge)는 사용자 인터랙션이 없으면 오디오 재생을 차단합니다.
`SoundProvider` 는 첫 `click`/`keydown`/`touchstart` 이벤트 후에만 엔진을 초기화하며,
그 이전의 재생 요청은 조용히 무시됩니다.
